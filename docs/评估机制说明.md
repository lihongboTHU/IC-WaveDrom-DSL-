## 一、 评估目标

本任务旨在评估多模态大模型将视觉波形图转换为结构化代码的能力。由于波形图包含复杂的时序逻辑、文本标签，传统的完全精准匹配无法真实反映大模型的视觉理解水平（例如轻微的拼写错误会导致整个样本计为 0 分）。

为此，这里设计了一套模拟了人类专家的阅卷思维综合打分机制，能从“信号名称（Name）”、“波形结构（Wave）”和“数据标签（Data）”三个维度进行细粒度的量化评估。

## 二、 评估流程与核心算法

评估系统分为 **数据修复** 与 **多维打分** 两个核心阶段：

### 2.1 结构化输出容错修复
由于大模型在生成超长时序波形时可能触发 `max_tokens` 截断，导致 JSON 括号未闭合。系统在评估前引入了自动修复函数。该层通过括号堆栈平衡算法，智能补全缺失的 `]`, `}`, 和 `"`，从而将截断的文本还原为合法的数据结构，确保大模型的视觉提取成果不因文本截断而被误判。

### 2.2 信号行对齐匹配
波形图在视觉上呈自上而下的列表结构。评估时，我们将真实标签与推理结果解析为有序列表。系统按照行的索引（第 $i$ 行对第 $i$ 行）进行匹配打分，这避免了因单行信号名称识别错误而导致整个波形匹配错乱的问题。

### 2.3 基于编辑距离的特征打分
对于信号的具体内容，系统采用基于编辑距离的相似度算法来计算匹配度，得分区间为 $[0.0, 1.0]$。该算法完美契合 OCR 与波形特征的评估需求：
* **对于文本 OCR**：能够合理包容个别字母的漏识别或错识别（例如识别 `SPISIMO` 为 `SPIMO`，相似度为 83.3% 予以保留部分分数）。
* **对于波形周期**：当模型正确识别了波形走势，但因周期比例拉伸导致字符串长度不同（如 `x====` 与 `x======`），算法依然能提取**最长公共子序列**，赋予合理的结构相似分。

## 三、 维度权重与分数计算

对于**对齐的每一行信号**，系统拆解出三个评估特征进行独立计算，并采取动态加权机制融合为单行最终得分。

1. **信号名称匹配度 ($Sim_{name}$)**：评估波形左侧标签的 OCR 准确度。
2. **波形结构匹配度 ($Sim_{wave}$)**：评估波形高低电平与时钟周期的正确性。
3. **数据标签匹配度 ($Sim_{data}$)**：针对波形中包含的数据文本。系统会先对预测列表与真实列表进行拼接，并剔除所有空格噪音，再进行相似度计算，消除大模型生成额外空格带来的干扰。

**单行动态权重公式：**
考虑到并非所有波形都包含数据标签（Data），权重的分配是动态自适应的：
* **场景 A：若该行波形无 Data 字段**
  波形走势是核心，名称为辅。
  $$Score_{row} = Sim_{name} \times 0.30 + Sim_{wave} \times 0.70$$
* **场景 B：若该行波形包含 Data 字段**
  波形结构与内部标签同等重要。
  $$Score_{row} = Sim_{name} \times 0.20 + Sim_{wave} \times 0.40 + Sim_{data} \times 0.40$$

## 四、 整体得分与长度惩罚机制

在得出所有对齐行的得分后，系统需计算整张图片的最终准确率。为了防止模型通过“少预测（漏行）”或“多预测（幻觉生成额外波形）”来取巧，系统引入了**最大行数惩罚机制**。

设真实标签行数为 $N_{gt}$，模型预测行数为 $N_{pred}$，最终准确率公式为：

$$ Accuracy_{overall} = \frac{\sum Score_{row}}{\max(N_{gt}, N_{pred})} $$

**机制优势解释：**
* **漏识别惩罚**：若图片中有 5 行波形，模型只生成了 3 行，分母将取 5。漏掉的 2 行被视为 0 分计入总分，严厉惩罚了信息遗漏。
* **过度生成惩罚**：若图片中只有 3 行，模型却产生了幻觉输出了 4 行。分母将取 4，多出的幻觉行稀释了总分，保证了评估的严谨性。
